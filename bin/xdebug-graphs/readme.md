## Usage

### Basic usage

**Important:** run from the folder that contains `cachegrind.*` files, or pass that path explicitly.

Use current directory (generates all types):
```bash
cd /path/to/xdebug-profiles
xdebug-graphs
```

Or pass a directory:
```bash
xdebug-graphs /path/to/xdebug-profiles
```

If you see `вњ… Done` with no `в†’ Generating` lines, the tool found no `cachegrind.*` files in that directory (wrong folder), or dependencies are missing (`pip install gprof2dot`, `brew install graphviz`).

### Generate specific types

Available types: `extra-small`, `small`, `medium`, `full`

| Type | Node Threshold | Edge Threshold | Use Case |
|------|----------------|----------------|----------|
| `extra-small` | 5% | 3% | Quick overview, main hotspots only |
| `small` | 2% | 1% | Balanced view |
| `medium` | 0.25% | 0.1% | Detailed analysis |
| `full` | 0.01% | 0.01% | Complete call graph |

Generate only specific types:
```bash
xdebug-graphs --types extra-small,small
xdebug-graphs /path/to/xdebug-profiles --types medium,full
xdebug-graphs --types small
```

If `--types` is not specified, all four types will be generated by default.

Existing PDFs are skipped (not overwritten).

## Output

Each generated PDF includes:

1. **Call graph** (Graphviz)
2. **Top N Blockera вЂ” Included Time**
3. **Top N Blockera вЂ” Self Time**
4. **Top N Blockera вЂ” Calls**
5. **Top N Non-Blockera вЂ” Included Time**
6. **Top N Non-Blockera вЂ” Self Time**
7. **Top N Non-Blockera вЂ” Calls**

Ranking tables use only the nodes present in that graph type (after the typeвЂ™s node/edge thresholds). Default is Top 20; `full` uses Top 60. Columns: Rank, Function/Method, Inc %, Self %, Calls.

Call graph styling:

- **Blue highlighted nodes**: Blockera-related functions
- **Red bordered nodes**: Top 5 most expensive Blockera functions (by included time)
- **Blue edges**: Calls originating from Blockera code
